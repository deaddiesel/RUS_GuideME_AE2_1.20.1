# ME Сундук

<GameScene zoom="8" background="transparent">
<ImportStructure src="../assets/blocks/chest.snbt" />
</GameScene>

ME Сундук действует как миниатюрная сеть с <ItemLink id="terminal" />, <ItemLink id="drive" />, и <ItemLink id="energy_acceptor" />.
Хотя его можно использовать как небольшую сеть хранения, его ёмкость всего для одной [ячейки хранения](../items-blocks-machines/storage_cells.md)
означает, что он имеет ограниченную пользу.

Вместо этого он полезен для взаимодействия с установленной внутри него ячейкой хранения. Его интегрированный ME Терминал может видеть и получать доступ только к предметам,
находящимся на установленном ME Дисководе, в то время как [устройства](../ae2-mechanics/devices.md) в общей сети могут получать доступ к предметам в любом [сетевом хранилище](../ae2-mechanics/import-export-storage.md),
включая ME Сундуки.

Он имеет два различных GUI и предназначен для транспортировки предметов. Взаимодействие с верхним ME Терминалом открывает интегрированный ME Терминал. Предметы можно вставлять в
установленную ячейку хранения через эту сторону, но нельзя извлекать. Взаимодействие с любой другой стороной открывает GUI со слотом для ячейки хранения
и настройками приоритета. Ячейку можно вставлять и извлекать с помощью логистики предметов только через сторону со слотом для ячейки.

Его можно повернуть с помощью <ItemLink id="certus_quartz_wrench" />.

У него небольшой буфер накопления энергии AE, поэтому, если он не подключен к сети с [энергохранилищем](../items-blocks-machines/energy_cells.md),
одновременная установка или извлечение слишком большого количества предметов может привести к его отключению.

Терминал можно раскрасить с помощью <ItemLink id="color_applicator" />.

## Настройки

У ME Сундука те же настройки, что и у <ItemLink id="terminal" /> или <ItemLink id="crafting_terminal" />.
Однако он не поддерживает <ItemLink id="view_cell" />.

## Светодиоды состояния ячейки

На ячейках в сундуке есть светодиод, который показывает их состояние:

| Цвет      | Статус                                                                                    |
| :-------- | :---------------------------------------------------------------------------------------- |
| Зелёный   | Пусто                                                                                     |
| Синий     | Имеет некоторое содержимое                                                                |
| Оранжевый | [Типы](../ae2-mechanics/bytes-and-types.md) заполнены, новые типы не могут быть добавлены |
| Красный   | [Байты](../ae2-mechanics/bytes-and-types.md) заполнены, больше нельзя добавить предметы   |
| Чёрный    | Нет энергии или ME Дисковода не имеющих [канал](../ae2-mechanics/channels.md)                |

## Приоритет

Приоритеты можно задать, щёлкнув значок гаечного ключа в правом верхнем углу GUI слота ячейки.
Предметы, поступающие в сеть, будут изначально попадать в хранилище с наивысшим приоритетом в качестве
первого места назначения. В случае двух хранилищ или ячеек с одинаковым приоритетом,
если одно из них уже содержит предмет, предпочтение будет отдано этому хранилищу.
Любые [разделённые](cell_workbench.md) ячейки будут рассматриваться как уже содержащие предмет,
если они находятся в той же группе приоритетов, что и другие хранилища. Предметы, удаляемые из хранилища, будут
удаляться из хранилища с самым низким приоритетом. Эта система приоритетов означает, что по мере добавления и удаления предметов
из сетевого хранилища, хранилища с более высоким приоритетом будут заполняться, а хранилища с более низким приоритетом будут освобождаться.
