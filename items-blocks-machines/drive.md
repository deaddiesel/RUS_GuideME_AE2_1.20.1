# ME Дисковод

<GameScene zoom="8" background="transparent">
  <ImportStructure src="../assets/blocks/drive.snbt" />
</GameScene>

ME Дисковод — это [устройство](../ae2-mechanics/devices.md), к которому вы подключаете [ячейки хранения](storage_cells.md), чтобы использовать их для
[сетевого хранилища](../ae2-mechanics/import-export-storage.md). Он имеет 10 слотов, каждый из которых вмещает одну ячейку.

Если по какой-то причине вы этого хотите, вы можете помещать и вытаскивать ячейки из инвентаря с помощью любого предмета логистики, например, воронок или ME Шин.

Его можно повернуть с помощью <ItemLink id="certus_quartz_wrench" />.

## Светодиоды состояния ячейки

На ячейках в сундуке есть светодиод, который показывает их состояние:

| Цвет      | Статус                                                                                    |
| :-------- | :---------------------------------------------------------------------------------------- |
| Зелёный   | Пусто                                                                                     |
| Синий     | Имеет некоторое содержимое                                                                |
| Оранжевый | [Типы](../ae2-mechanics/bytes-and-types.md) заполнены, новые типы не могут быть добавлены |
| Красный   | [Байты](../ae2-mechanics/bytes-and-types.md) заполнены, больше нельзя добавить предметы   |
| Чёрный    | Нет энергии или ME Дисковода не имеющих [канал](../ae2-mechanics/channels.md)                |

## Приоритет

Приоритеты можно задать, щёлкнув значок гаечного ключа в правом верхнем углу GUI слота ячейки.
Предметы, поступающие в сеть, будут изначально попадать в хранилище с наивысшим приоритетом в качестве
первого места назначения. В случае двух хранилищ или ячеек с одинаковым приоритетом,
если одно из них уже содержит предмет, предпочтение будет отдано этому хранилищу.
Любые [разделённые](cell_workbench.md) ячейки будут рассматриваться как уже содержащие предмет,
если они находятся в той же группе приоритетов, что и другие хранилища. Предметы, удаляемые из хранилища, будут
удаляться из хранилища с самым низким приоритетом. Эта система приоритетов означает, что по мере добавления и удаления предметов
из сетевого хранилища, хранилища с более высоким приоритетом будут заполняться, а хранилища с более низким приоритетом будут освобождаться.
